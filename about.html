The Vinyl Scout
Collection Add Audit

How it works · v22 · July 2026

A pocket catalog for a personal record collection.

The Vinyl Scout is a small, opinionated web app for tracking one person's vinyl LPs — what's owned, what condition it's in, what each pressing is worth on the secondary market, and which exact edition you own. Powered by Discogs API enrichment for accurate pricing and pressing data. No accounts, no third-party tracking, no automations that quietly delete data.

01What it is

A single-tenant catalog of physical records. The owner photographs covers in small batches, Claude identifies artist and title from each cover, and a short list of Artist - Title lines gets pasted into /seed.html to register the records. Every entry is editable on /audit.html.

Each record is enriched with Discogs data: accurate pressing ID, marketplace prices (low, median, high from sales history), community ownership and wishlist counts, and ratings. Prices refresh monthly via a batch script.

The collection page itself — what you came from — is read-only: a searchable, genre-filterable list or gallery. Tap any record for the detail view, which shows the full Discogs marketplace snapshot.

02The stack

  • Frontend. Static HTML, CSS, and one JavaScript file. No framework, no build step. Served from Netlify's CDN.
  • Storage. A single Netlify Blobs store called records. One JSON blob per record, keyed by ID.
  • API. Netlify Functions v2 for GET (list all records), POST (upsert by ID), DELETE (single record). Enrichment endpoints: /api/discogs/lookup (candidate matching by artist/title), and /api/discogs-pricing (fetch + store pricing & metadata). All write endpoints require an X-Edit-Key header.
  • Source. Plain Git repo at github.com/snesbitt/vinyl-scout. Every change ships via git push; Netlify rebuilds in under a minute.
  • Authoring. Code and copy written collaboratively with Claude; deploys reviewed and run by the owner.

03The catalog record

Every record is one JSON object. Phase 1 fields are the originals; Phase 2 (live) adds Discogs enrichment: condition, accurate pressing ID, marketplace pricing, community data, and ratings.

  • id — opaque 16-character hex string (e.g. rec_a1b2c3d4e5f60718).
  • artist, title — required strings.
  • year, genre — optional. Year is 1900–2100.
  • cover_url — optional image URL or inline data URL.
  • notes — optional free text.
  • condition — Goldmine grade (default VG). See below.
  • price_low, price_median, price_high — market prices in price_currency, optional. Low is the current cheapest listing; median and high are from the release's last-30-sales history.
  • price_last_sold — string date of the most recent completed sale (e.g. "Apr 23, 2026"), or null if the release has never sold.
  • copies_available — count of copies currently for sale, optional.
  • have_count, want_count — Discogs community ownership and wishlist counts, optional.
  • rating_avg, rating_count — Discogs community rating (0–5) and number of ratings, optional.
  • price_currency — ISO code (USD, EUR, GBP), optional.
  • price_updated_at — ISO timestamp of last refresh, optional.
  • discogs_release_id — cached release ID for fast refreshes, optional.

04Condition grades

Vinyl Scout uses Goldmine grading (the US standard for record sleeves and discs). Each record stores its own condition code. Default is VG (Very Good).

  • M — Mint. Never played, still sealed or in perfect condition.
  • NM — Near Mint. Played once or twice, looks and sounds unplayed.
  • VG+ — Very Good Plus. Minor wear; fully playable with no defects.
  • VG — Very Good. Light surface marks, plays without issue.
  • G+ — Good Plus. Noticeable wear, plays through, some surface noise.
  • G — Good. Significant wear, plays but with some noise.
  • F — Fair. Heavy wear, plays but may skip or have extended noise.
  • P — Poor. Severely damaged, barely playable.

05Data & backups

Every record is backed up automatically. A nightly function commits the entire catalog to backups/YYYY-MM-DD.json in the GitHub repo. You can also trigger a backup on demand from the API.

The catalog is read-only for the public (anyone can browse). Edits require a shared secret key, sent as a request header. There are no user accounts.
